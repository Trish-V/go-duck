Secured Kratos gRPC APIs

Access the generated microservices over ultra-fast gRPC using Kratos framework, fully authenticated by Keycloak.

Overview

The GO-DUCK CLI generated app binds the Gin REST webserver on port 8080 and simultaneously spins up an optimized Go-Kratos gRPC listener on port 9000. Both the REST and gRPC handlers share the exact same clean Repository architecture underneath, guaranteeing 100% data and logic consistency.

Note: api/v1/*.proto definitions are generated automatically whenever you add an entity or change fields.

Kratos JWT Middleware Security

By default, Kratos attaches a strict validation middleware out of the box. Anyone trying to consume your internal services via gRPC must be authenticated using your OIDC provider.

Client Connection Example (Go)

This snippet demonstrates how a developer orchestrating another microservice connects securely using Google's `oauth.TokenSource` to fulfill the Kratos Auth Header standards.

import (
    "context"
    "log"
    "google.golang.org/grpc"
    "golang.org/x/oauth2"
    "google.golang.org/grpc/credentials/oauth"
    pb "go-duck/api/v1"
)

func main() {
    // Inject the valid Keycloak JWT token securely
    token := oauth.TokenSource{
        TokenSource: oauth2.StaticTokenSource(&oauth2.Token{
            AccessToken: "eyJh...", 
        }),
    }

    // Dial the local port 9000 (GoDuck Server)
    conn, err := grpc.Dial("localhost:9000", grpc.WithInsecure(), grpc.WithPerRPCCredentials(token))
    if err != nil {
        log.Fatalf("did not connect: %v", err)
    }
    defer conn.Close()

    // Initialize the Kratos-generated gRPC Client
    client := pb.NewEntityServiceClient(conn)

    // Execute Call Native Network Call
    res, err := client.GetEntity(context.Background(), &pb.GetEntityRequest{Id: 1})
    if err != nil {
        log.Fatalf("RPC failed: %v", err)
    }
    
    log.Printf("Successfully Received Data Pipeline Object OVER gRPC: %v", res)
}
