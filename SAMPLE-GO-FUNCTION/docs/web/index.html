go-duck Developer Guide

Welcome to the developer documentation for your GO-DUCK generated microservice. This guide covers how to connect, authenticate, and integrate with all the implemented features (The 200% Milestone).

Quick Start & Setup

Note: Most APIs are secured by Keycloak JWT. You must start the infrastructure to have access to Keycloak, DB, Redis, etc.

Running the application locally:

# Start local infrastructure (Postgres, Redis, Mosquitto MQTT, Keycloak, Jaeger, OTel Collector)
docker-compose up -d

# Install dependencies and start the Go server
go mod tidy
go run main.go

Features Overview:

  • Full CRUD REST APIs & Search: Standard routes and a PostgREST-like RPC generic query system.
  • GraphQL: A complete, connected GraphQL schema integrated directly with the GORM models.
  • Secure Kratos gRPC APIs: Ultra-fast Protobuf services automatically configured with Keycloak JWT authentication.
  • Real-Time Operations: REST-over-WebSockets dispatcher with HMAC-SHA256 integrity and async MQTT Pub/Sub events.
  • Auditing & Metering: Automatic interceptors for tracking mutation footprints and managing API usage limits.
  • Infrastructure Ready: OpenTelemetry integration, Dockerization, and automated incremental Liquibase migrations.
