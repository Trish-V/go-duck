Standard Controller Features

📦 Eager vs. Lazy Loading

By default, entities are loaded Lazily (IDs only for foreign keys). To perform a Join and fetch the full object graph, use the eager parameter.

Lazy (Default)

GET /api/article/1

Eager (Joined)

GET /api/article/1?eager=true

📏 Offset Pagination

All collection endpoints support built-in pagination to keep your frontend lightning fast.

Distributed Redis Shield

Every GetByID request is automatically protected by a Redis cache layer. Mutations (Create/Update/Delete) trigger Automatic Cache Invalidation across the cluster, ensuring data consistency with zero effort.

Deep JSON/JSONB Querying

The GO-DUCK generator natively supports PostgreSQL JSONB operators. You can drill down into nested fields directly from the URL using arrow notation.

Text Extraction (->>) Standard Use
GET /api/rpc/users?metadata->>role=eq.ADMIN

Extracts the value as text. Perfect for equality checks on nested strings.

Object Extraction (->)
GET /api/rpc/orders?details->itemsCount=gt.5

Treats the extracted value as a JSON object/numeric, allowing for range checks on nested numbers.

Pro Tip: For high-performance JSON querying, ensure you have a GIN index on the JSONB column in your database.

Bulk Mission Control (High Velocity)

For batch processing and migrations, avoid the overhead of multiple HTTP calls. Use the specialized /bulk endpoints to process hundreds of records in a single transaction.

Bulk Create Transaction POST /api/:entities/bulk
